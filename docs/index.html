Обзор

Проект реализует игру «Сапёр» на Qt/QML. Код разделён на несколько слоёв:

  • core — игровая логика, хранение поля и модель для отображения поля.
  • app::managers — менеджеры приложения, связывающие C++-логику с QML.
  • utility — небольшие вспомогательные функции.

Архитектура

Основной игровой слой состоит из следующих классов:

  • core::MinefieldInteractor хранит параметры партии: ширину, высоту и количество мин.
  • core::MinefieldStorage хранит клетки в линейном массиве и умеет обходить соседей.
  • core::Minefield содержит правила игры: генерацию поля, открытие клеток, флаги и проверку победы.
  • core::MinefieldModel адаптирует поле под QML через QAbstractItemModel и роли модели.

Слой менеджеров создаёт эти объекты, регистрирует их в QQmlApplicationEngine и управляет настройками, ресурсами и текстурными наборами.

Связь с QML

Объекты, наследующие QObject, публикуют данные через Q_PROPERTY и Q_INVOKABLE. Основные точки входа для интерфейса:

  • app::managers::GameInteractor — запуск игры и передача кликов.
  • app::managers::SessionInteractor — настройка размеров поля и количества мин.
  • app::managers::SettingsInteractor — параметры масштаба и громкости.
  • app::managers::TextureInteractor — пути к изображениям текущего набора текстур.
  • core::MinefieldModel — данные клеток и камера доски.

Ресурсы

app::managers::ResourceManager использует FileSetLoader для поиска наборов ресурсов рядом с исполняемым файлом в директории resources. Описание обязательных файлов находится в app::managers::file_collector::collection.
